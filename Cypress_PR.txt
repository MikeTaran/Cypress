JenkinsQA_JS_06
работа с проектом
1. Получить приглашения в Trello и GitHub
2. Склонировать репозиторий согласно инструкции в readme.md
3. В Trello в столбце US_ToDo выбрать userStory: US_01.02
4. Создать карточку тест-кейса TC из шаблона в backloge, дав ему имя:
	ТС_01.02.021 | searchboxPlaceholderCheck, где 01.02 - номер из US, 021 - номер следующего ТС в этой US
5. Создать карточку автотеста АT из шаблона в backloge, дав ему имя:
	АТ_01.02.021 | searchboxPlaceholderCheck, где 01.02 - номер из US, 021 - номер следующего ТС в этой US
6. В каждой карточке добавить себя как создателя, нажав Join и связать карточки между собой через Attach/Trello
7. Расписать ТС согласно условия: 1 ТС одна проверка
8. Если для этой US уже писался автотест, то можно к ниму присоединиться, добвивив свой IT
9. Открываем VSC и делаем git pull, затем создаем новую ветку с именем:
	<инициалы>/<ТС description> : mt/searchboxPlaceholderCheck
10. Открываем существующий(или создаем новый) автотест и добавляет свою проверку, дав имя IT как название AT
11. Во вкладке гита нажимаем + справа от имени файла, добавляем коммит по имени АТ и commit.
12. Создаем Pull request:
	git push
13. Переходим в гитхаб и на вкладке Pull request нажимаем ссобщение о создании нового PR.
14. D описание добавляем url нашего АТ из Trello и создаем PR
15. Ссылку на PR attach в AT
16. Тест должен пройти CI валидацию и получить 2 апрува
17. Нажитмаем Squash and merge, редактируем коммиты и подтверждаем
18. При удачном мерже удаляем ветку в гите
19. Переходим в VSC, в main ветку и делаем git pull, после этого удаляем ветку с тестом.
